# 系统架构设计文档

## 一、系统概述

### 1.1 项目定位
兰州财经大学周边公共建筑屋顶光伏潜力评估与碳减排决策平台是一个综合性的数据可视化与分析系统，旨在为校园及周边区域的光伏建设提供科学决策支持。

### 1.2 设计理念
- **数据驱动**：以GIS和数据分析为核心
- **可视化优先**：直观的图表和地图展示
- **响应式设计**：适配多种设备和屏幕尺寸
- **模块化架构**：便于功能扩展和维护

## 三、功能模块设计

### 3.1 首页 (Home)
**功能定位**：系统概览和数据仪表盘

**核心功能**：
- 地图展示（GIS集成）
- 核心指标统计卡片
- 评估流程展示
- 数据图表分析
- 实时数据刷新
- 快捷入口

### 3.2 适宜性评估 (Suitability)
**功能定位**：建筑光伏安装适宜性分析

**核心功能**：
- 多因子分析模型
  - 屋顶面积因子
  - 朝向角度因子
  - 遮挡情况因子
  - 结构承载因子
  - 周边环境因子
- 适宜性分级（5级）
  - 非常适宜（≥90分）
  - 较适宜（80-90分）
  - 一般（60-80分）
  - 较不适宜（40-60分）
  - 不适宜（<40分）
- 空间分布热力图
- 单体建筑详情查询
- 批量评估报告导出

### 3.3 效益评估 (Benefit)
**功能定位**：经济效益分析

**核心功能**：
- 装机容量估算
- 年发电量预测
  - 月度发电曲线
  - 季节性分析
  - 多年趋势预测
- 投资成本分析
  - 设备成本
  - 安装成本
  - 运维成本
- 收益测算
  - 电费节省
  - 补贴收益
  - 碳交易收益
- 投资回收期计算
- 敏感性分析
- 经济效益对比

### 3.4 碳减排评估 (Emission)
**功能定位**：环境效益分析

**核心功能**：
- 碳减排量核算
  - 年度减排量
  - 累计减排量
  - 减排贡献排名
- 环境效益折算
  - 标准煤节约
  - SO₂减排
  - NOₓ减排
  - 粉尘减排
- 碳排放权交易
  - CCER项目潜力
  - 碳配额估算
- ESG报告生成
- 碳中和路径规划

### 3.5 决策支持 (Decision)
**功能定位**：综合决策分析

**核心功能**：
- 多目标优化模型
- 优先级排序算法
- 推荐方案生成
- 情景模拟分析
- 投资组合优化
- 风险评估矩阵
- 决策报告导出

### 3.6 可视化分析 (Visual)
**功能定位**：高级数据可视化

**核心功能**：
- 3D场景展示
  - 建筑模型展示
  - 光伏板铺设模拟
  - 光照分析可视化
- 时空演变动画
- 多维数据对比
- 自定义图表配置
- 数据大屏模式
- 全屏展示功能

### 3.7 运维管理 (Operation)
**功能定位**：系统运维支持

**核心功能**：
- 数据管理
  - 数据导入/导出
  - 数据质量监控
  - 历史数据查询
- 任务调度
  - 评估任务配置
  - 自动报告生成
  - 定时数据更新
- 预警管理
  - 设备状态监控
  - 异常告警
  - 维护提醒
- 日志管理
  - 操作日志
  - 系统日志
  - 审计追踪

### 3.8 系统管理 (System)
**功能定位**：系统配置管理

**核心功能**：
- 用户管理
  - 用户列表
  - 角色权限
  - 部门管理
- 角色权限
  - 管理员
  - 操作员
  - 访客
- 系统配置
  - 参数设置
  - 阈值配置
  - 地图配置
- 备份还原
  - 数据备份
  - 配置导出
  - 灾难恢复

## 四、数据模型设计

### 4.1 建筑数据模型
```javascript
{
  id: String,           // 建筑唯一标识
  name: String,         // 建筑名称
  type: String,         // 建筑类型
  location: {
    lat: Number,        // 纬度
    lng: Number,        // 经度
    address: String     // 地址
  },
  roofArea: Number,     // 屋顶面积(m²)
  usableArea: Number,   // 可利用面积(m²)
  orientation: Number,  // 朝向角度(°)
  tilt: Number,         // 倾斜角度(°)
  structure: String,    // 结构类型
  yearBuilt: Number,    // 建造年份
  floors: Number,       // 楼层数
  suitability: {
    score: Number,      // 适宜性评分
    level: String,      // 适宜性等级
    factors: Object      // 各因子得分
  },
  potential: {
    installCapacity: Number,  // 装机容量(kW)
    annualGeneration: Number, // 年发电量(kWh)
    carbonReduction: Number,   // 年减碳量(tCO₂)
    investment: Number,       // 投资估算(万元)
    paybackPeriod: Number     // 回收期(年)
  },
  status: String,       // 状态
  createdAt: Date,      // 创建时间
  updatedAt: Date       // 更新时间
}
```

### 4.2 评估任务模型
```javascript
{
  id: String,
  name: String,
  type: String,         // evaluation | simulation | report
  status: String,       // pending | running | completed | failed
  parameters: Object,
  results: Object,
  createdBy: String,
  createdAt: Date,
  completedAt: Date
}
```

## 五、API接口设计

### 5.1 建筑接口
| 方法 | 路径 | 描述 |
|------|------|------|
| GET | /api/buildings | 获取建筑列表 |
| GET | /api/buildings/:id | 获取建筑详情 |
| POST | /api/buildings | 新增建筑 |
| PUT | /api/buildings/:id | 更新建筑 |
| DELETE | /api/buildings/:id | 删除建筑 |
| POST | /api/buildings/evaluate | 批量评估 |

### 5.2 评估接口
| 方法 | 路径 | 描述 |
|------|------|------|
| GET | /api/evaluations | 获取评估记录 |
| POST | /api/evaluations/suitability | 适宜性评估 |
| POST | /api/evaluations/benefit | 效益评估 |
| POST | /api/evaluations/emission | 碳减排评估 |

### 5.3 系统接口
| 方法 | 路径 | 描述 |
|------|------|------|
| GET | /api/stats/overview | 获取概览数据 |
| GET | /api/stats/trends | 获取趋势数据 |
| POST | /api/reports/generate | 生成报告 |

## 六、响应式设计

### 6.1 断点设计
| 设备 | 宽度 | 布局 |
|------|------|------|
| 超大屏 | ≥1920px | 4列布局，侧边栏展开 |
| 大屏 | 1440-1919px | 3列布局，侧边栏展开 |
| 中屏 | 1024-1439px | 2列布局，侧边栏收起 |
| 平板 | 768-1023px | 单列布局，底部导航 |
| 移动端 | <768px | 全屏单列，手势导航 |

### 6.2 组件适配策略
- **Header**：移动端折叠为汉堡菜单
- **Sidebar**：大屏横向展示，移动端收起
- **MainContent**：弹性布局，自适应宽度
- **SidePanel**：大屏展示，移动端隐藏或底部抽屉
- **Charts**：响应式尺寸，触摸友好
